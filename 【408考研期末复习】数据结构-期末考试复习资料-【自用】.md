## 数据结构期末考试复习资料

### 一、绪论

#### 1.1 基本概念

✨✨基本概念两颗星

* **数据（Data）**：对客观事物的符号表示，是计算机程序处理的对象，涵盖数值、字符、图像、声音等多种形式的信息。
* **数据元素（Data Element）**：作为数据的基本单位，也被称作数据项或记录。它通常由若干个不可分割的最小单位 —— 数据项组成。 我们最熟悉的excel中学生信息表的一条条数据。
* **数据对象（Data Object）**：性质相同的数据元素的集合，属于数据的一个子集。
* **数据结构（Data Structure）**：指相互之间存在一种或多种特定关系的数据元素的集合。这些关系包括集合关系、线性关系、树形关系或图形关系等。数据结构一般包含**数据的逻辑结构、存储结构和操作**三个方面。 考试考填空题，线性、树形、图形关系

#### 1.2 逻辑结构

1. **集合结构**：数据元素之间仅存在 “同属于一个集合” 的关系，除此之外无其他特定关系。
2. **线性结构**：数据元素呈现一对一的关系，构成有序序列，如链表、队列、栈等。
3. **树形结构**：数据元素具有一对多的层次关系，像**二叉树、二叉查找树、平衡二叉查找树**等。
4. **图状（网状）结构**：数据元素间存在多对多的复杂关系。

#### 1.3 存储结构（物理结构）

1. **顺序存储结构**：借助一段物理地址连续的存储单元依次存储数据元素，通常采用数组实现。该结构可随机存取，存储密度高，但插入和删除操作可能需要移动大量元素。
2. **链式存储结构**：通过指针将离散分布的结点连接起来，每个结点包含数据域和指针域。其插入和删除操作较为灵活，无需大量移动元素，但存储密度较低，不支持随机存取。**物理上是不相连的，逻辑上是相连的**
3. **索引存储结构**：建立附加的索引表，索引表中的索引项与数据元素一一对应，通过索引表可快速定位数据元素，提升查找效率，但会占用额外空间。
4. **哈希存储结构**：依据哈希函数，将数据元素的关键字映射到存储地址。哈希存储具有高效的查找、插入和删除操作性能，但可能出现哈希冲突，需采用合适的冲突处理方法。

#### 1.4 算法

1. **定义**：对特定问题求解步骤的描述，是指令的有限序列，每条指令包含一个或多个操作。
2. **特性**

* **有穷性**：算法执行的步骤必须是有限的，在有限时间内能够结束。
* **确定性**：算法的每一步骤都有明确的定义，不存在二义性。
* **可行性**：算法的所有操作都可通过已实现的基本运算执行有限次来完成。
* **输入**：算法有零个或多个输入，这些输入取自特定的对象集合。
* **输出**：算法有一个或多个输出，用于反映对输入数据加工后的结果。

这部分选择题和解答题考烂了好吧

1. **性能指标**

* **正确性**：算法应能正确解决给定的问题，对于合法输入能得到正确的输出结果。
* **可读性**：算法易于理解和阅读，便于交流和维护。
* **健壮性**：算法对非法输入具有良好的容错能力，不会因非法输入而异常终止，能给出适当的提示或处理。
* **高效率与低存储需求**：时间复杂度低，执行效率高；空间复杂度低，占用存储空间少。

1. **时间复杂度分析**：算法执行时间与问题规模 n 相关，常见的时间复杂度有 O (1)（常数阶）、O (n)（线性阶）、O (n²)（平方阶）、O (log n)（对数阶）、O (n log n)（线性对数阶）等。通过分析算法中基本操作的执行次数来确定时间复杂度，例如：

```
// 时间复杂度为O(n)的示例

for (i = 0; i < n; i++) {

&#x20;   // 基本操作

}

```

```
// 时间复杂度为O(n²)的示例

for (i = 0; i < n; i++) {

&#x20;   for (j = 0; j < n; j++) {

&#x20;       // 基本操作

&#x20;   }

}

```

1. **空间复杂度分析**：算法执行过程中所需的辅助空间大小与问题规模 n 的关系。若算法执行所需的辅助空间为常量，与问题规模 n 无关，则空间复杂度为 O (1)，如：

```
// 空间复杂度为O(1)的示例

int a = 0;

int b = 1;

int c = a + b;

```

若算法需要动态分配与问题规模 n 相关的空间，则空间复杂度为 O (n)，如：

```
// 空间复杂度为O(n)的示例

int \*arr = (int \*)malloc(n \* sizeof(int));

```

### 二、线性表

#### 2.1 定义与基本操作

1. **定义**：线性表是 n 个具有相同特性的数据元素的有限序列。在逻辑上呈线性结构，物理存储时通常采用数组或链式结构。
2. **基本操作**

* **初始化**：创建一个空的线性表。
* **插入**：在指定位置插入一个数据元素。
* **删除**：删除指定位置的数据元素。
* **查找**：根据给定值查找数据元素的位置。
* **修改**：修改指定位置的数据元素值。
* **求长度**：返回线性表中数据元素的个数。

#### 2.2 顺序存储结构

1. **特点**：用一段物理地址连续的存储单元依次存储数据元素，可实现随机存取，即通过数组下标能直接访问任意元素。但插入和删除操作可能需要移动大量元素，时间复杂度为 O (n)。
2. **实现**：使用数组实现顺序表，例如：

```
define MAXSIZE 100 // 定义顺序表的最大长度

typedef struct {

  int data[MAXSIZE]; // 存储数据元素的数组

   int length; // 顺序表当前长度

} SqList;

```

1. **插入操作**：在顺序表的第 i 个位置插入元素 e，需将第 i 个位置及之后的元素依次向后移动一位，然后将 e 插入到第 i 个位置。若 i 位置不合法或顺序表已满，则插入失败。示例代码如下：

```
int ListInsert(SqList *L, int i, int e) {

  if (i < 1 || i > L->length + 1) // 检查插入位置是否合法

     return 0;

 if (L->length >= MAXSIZE) // 检查顺序表是否已满

     return 0;

   for (int j = L->length; j >= i; j--)

       L->data[j] = L->data[j - 1]; // 元素后移

   L->data[i - 1] = e; // 插入元素

   L->length++; // 顺序表长度加1

  return 1;

}

```

1. **删除操作**：删除顺序表中第 i 个位置的元素，需将第 i + 1 个位置及之后的元素依次向前移动一位，顺序表长度减 1。若 i 位置不合法，则删除失败。示例代码如下： 这其实和插入的逻辑是相似的。

```
int ListDelete(SqList *L, int i, int *e) {  // 这里的指针是保存要删除的数据元素，因为不保存的话就真的物理删除了，其实但大部分是删除都是逻辑删除

  if (i < 1 || i > L->length) // 检查删除位置是否合法

      return 0;

   *e = L->data[i - 1]; // 保存被删除元素

  for (int j = i; j < L->length; j++)

       L->data[j - 1] = L->data[j]; // 元素前移

  L->length--; // 顺序表长度减1

  return 1;

}

```

#### 2.3 链式存储结构

1. **单链表**

* **特点**：由一系列结点组成，每个结点包含数据域和指针域，指针域指向下一个结点。存储不连续，非随机存取，插入和删除操作只需修改指针，无需移动大量元素，时间复杂度为 O (1)（已知插入或删除位置的前驱结点时）。
* **实现**：定义单链表结点结构，例如：

```
typedef struct LNode {

   int data; // 数据域

   struct LNode *next; // 指针域

} LNode, *LinkList;

```

* **头插法建立单链表**：从一个空链表开始，每次将新结点插入到链表头部，新插入的结点成为新的头结点。示例代码如下：

```
LinkList CreateListHead(LinkList &L) {

 L = new LNode;

   L->next = NULL; // 初始化头结点

   int x;

   scanf("%d", &x); // 输入数据，假设以-1作为结束标志

  while (x != -1) {

     LNode *s = new LNode;

     s->data = x;

      s->next = L->next;

     L->next = s; // 将新结点插入到头部

     scanf("%d", \&x);

  }

 return L;

}

```

* **尾插法建立单链表**：从一个空链表开始，设置一个尾指针始终指向链表尾部，每次将新结点插入到链表尾部。示例代码如下：

```
LinkList CreateListTail(LinkList &L) {

  L = new LNode;

 L->next = NULL;

   LNode *r = L; // r为尾指针，初始指向头结点

   int x;

   scanf("%d", \&x);

   while (x != -1) {

      LNode *s = new LNode;

     s->data = x;

      r->next = s;

      r = s; // 将新结点插入到尾部，更新尾指针

       scanf("%d", \&x);

 }

   r->next = NULL; // 尾指针的next置空

  return L;

}

```

* **插入操作**：在单链表中第 i 个位置插入元素 e，需先找到第 i - 1 个结点，然后创建新结点，将新结点插入到第 i - 1 个结点之后。示例代码如下：

```
int ListInsert(LinkList &L, int i, int e) {

  LNode *p = L;

   int j = 0;

  while (p && j < i - 1) {

      p = p->next;

      j++;

   } // 查找第i - 1个结点

  if (!p || j > i - 1) // i大于表长或i小于1

     return 0;

  LNode *s = new LNode;

   s->data = e;

  s->next = p->next;

  p->next = s; // 插入新结点

   return 1;

}

```

* **删除操作**：删除单链表中第 i 个位置的元素，需先找到第 i - 1 个结点，然后修改指针，跳过第 i 个结点。示例代码如下：

```
int ListDelete(LinkList &L, int i, int &e) {

  LNode *p = L;

  int j = 0;

   while (p->next && j < i - 1) {

     p = p->next;

      j++;

   } // 查找第i - 1个结点

   if (!(p->next) || j > i - 1) // i大于表长或i小于1

       return 0;

   LNode *q = p->next;

  e = q->data;

   p->next = q->next;

  delete q; // 删除结点

  return 1;

}

```

1. **循环单链表**：与单链表的区别在于，循环单链表的尾结点的指针指向头结点，形成一个环形结构。从任意结点出发都能遍历整个链表。
2. **双向链表**：每个结点包含数据域、前驱指针域和后继指针域，可双向遍历链表，插入和删除操作更加灵活，但增加了空间开销。双向链表结点结构定义如下：

```
typedef struct DuLNode {

   int data;

  struct DuLNode *prior;

  struct DuLNode *next;

} DuLNode, *DuLinkList;

```

双向链表的插入和删除操作示例代码如下：

```
// 双向链表插入操作

int DuListInsert(DuLinkList &L, int i, int e) {

   DuLNode *p = L;

   int j = 0;

   while (p && j < i - 1) {

      p = p->next;

       j++;

   } // 查找第i - 1个结点

   if (!p || j > i - 1)

      return 0;

   DuLNode *s = new DuLNode;

 s->data = e;

  s->prior = p;

  s->next = p->next;

  if (p->next)

     p->next->prior = s;

 p->next = s; // 插入新结点

  return 1;

}

```

```
// 双向链表删除操作

int DuListDelete(DuLinkList &L, int i, int &e) {

   DuLNode *p = L;

   int j = 0;

  while (p->next && j < i - 1) {

      p = p->next;

      j++;

  } // 查找第i - 1个结点

  if (!(p->next) || j > i - 1)

       return 0;

   DuLNode \*q = p->next;

   e = q->data;

   p->next = q->next;

  if (q->next)

      q->next->prior = p;

   delete q; // 删除结点

   return 1;

}

```

1. **静态链表**：用数组模拟链表，逻辑上像**链表，物理上是数组**。每个数组元素包含**数据域和指针域**（这里的指针是数组下标）。静态链表适用于不支持指针的编程语言或对内存管理有特殊要求的场景。静态链表结构定义如下：

```
define MAXSIZE 100

typedef struct {

  int data;

  int cur; // 游标（代替指针）

} SLinkList[MAXSIZE];

```

初始化静态链表及相关操作示例代码如下：

```
// 初始化静态链表

void InitList(SLinkList &space) {

   for (int i = 0; i < MAXSIZE - 1; i++)

       space[i].cur = i + 1;

   space[MAXSIZE - 1].cur = 0; // 最后一个元素的cur为0，表示链表结束

}

```

```
// 分配一个结点

int Malloc(SLinkList &space) {

  int i = space[0].cur;

   if (space[0].cur)

       space[0].cur = space[i].cur;

  return i;

}

```

```
// 回收一个结点

void Free(SLinkList &space, int k) {

  space[k].cur = space[0].cur;

   space[0].cur = k;

}

```

### 三、栈与队列（续）

#### 3.1 栈（续）

3. **链式存储结构实现（链栈）**

   * **定义**：用单链表实现，通常以链表头部作为栈顶（减少插入/删除操作的复杂度），无需预设容量，避免栈满问题。链栈结点结构与单链表一致，定义如下：

   ```
   typedef struct StackNode {
       int data;                  // 数据域
       struct StackNode *next;    // 指针域，指向栈底方向的下一个结点
   } StackNode, *LinkStack;

   ```

   * **核心操作**
     + **初始化**：创建空栈，栈顶指针置空。

     ```
     void InitStack(LinkStack &S) {
         S = NULL;  // 栈空时，栈顶指针为NULL
     }

     ```

     + **判断栈空**：栈顶指针为NULL即栈空。

     ```
     int StackEmpty(LinkStack S) {
         return S == NULL ? 1 : 0;
     }

     ```

     + **入栈**：新结点插入栈顶（链表头部）。

     ```
     int Push(LinkStack &S, int e) {
         StackNode *p = new StackNode;  // 新建结点
         p->data = e;
         p->next = S;  // 新结点指向原栈顶
         S = p;        // 栈顶指针更新为新结点
         return 1;
     }

     ```

     + **出栈**：删除栈顶结点并返回其值。

     ```
     int Pop(LinkStack &S, int &e) {
         if (StackEmpty(S)) return 0;  // 栈空，出栈失败
         StackNode *p = S;              // 暂存栈顶结点
         e = p->data;                   // 保存栈顶元素值
         S = S->next;                   // 栈顶指针下移
         delete p;                      // 释放原栈顶结点
         return 1;
     }
     // 和逻辑出栈是先指针下移再删除原来栈顶的指针

     ```

     + **取栈顶元素**：仅读取栈顶值，不删除结点。

     ```
     int GetTop(LinkStack S, int &e) {
         if (StackEmpty(S)) return 0;  // 栈空，无栈顶元素
         e = S->data;
         return 1;
     }

     ```
   * **特点**：动态扩容，无栈满问题；插入/删除时间复杂度O(1)，但因指针域存在，存储密度低于顺序栈。
4. **栈的典型应用**

   * **括号匹配**：遍历表达式，遇到左括号入栈，遇到右括号则出栈顶左括号并检查匹配（如`(`对应`)`、`[`对应`]`），最终栈空且无不匹配则合法。
   * **表达式求值**：将中缀表达式转为后缀表达式（利用栈处理运算符优先级），再通过栈计算后缀表达式结果（遇到数字入栈，遇到运算符弹出两个数计算后入栈）。 后缀表达式类似于 二叉树的后序遍历
   * **函数调用与递归**：程序执行时用“调用栈”保存函数返回地址、局部变量，递归深度本质是栈的最大占用深度（递归过深会导致栈溢出）。  
     ***这里一般说是考选择题，还有填空题，两分***

#### 3.2 队列

1. **定义与特点**：先进先出（FIFO，First In First Out）结构，限定在一端（队尾）插入、另一端（队头）删除。常用于任务调度、缓冲处理（如打印机队列）、BFS（广度优先搜索）等场景。 高选择题，打印机就是类似于队列，进程中按照循序等待任务执行调度。
2. **顺序存储结构实现（顺序队列）**

   * **问题**：直接用数组实现时，队头删除元素后会导致“假溢出”（数组尾部有空间但无法利用），需引入“循环队列”解决。
   * **循环队列**：将数组视为环形，队头、队尾指针超过数组边界后自动指向起始位置，通过“\*\*队尾指针+1 == 队头指针”\*\*判断栈满（需牺牲一个位置区分栈满与栈空）。
     + **定义**：

     ```
     #define MAXSIZE 100
     typedef struct {
         int data[MAXSIZE];  // 存储队列元素
         int front;          // 队头指针（指向队头元素）
         int rear;           // 队尾指针（指向队尾元素的下一个位置）
     } SqQueue;

     ```

     + **核心操作**
       - **初始化**：队头、队尾指针均置0。

       ```
       void InitQueue(SqQueue &Q) {
           Q.front = Q.rear = 0;
       }

       ```

       - **判断队列空/满**：

       ```
       // 队列空：队头==队尾
       int QueueEmpty(SqQueue Q) {
           return Q.front == Q.rear ? 1 : 0;
       }
       // 队列满：(队尾+1)%MAXSIZE == 队头
       int QueueFull(SqQueue Q) {
           return (Q.rear + 1) % MAXSIZE == Q.front ? 1 : 0;
       }

       ```

       - **入队**：队尾指针后移（环形），插入元素。

       ```
       int EnQueue(SqQueue &Q, int e) {
           if (QueueFull(Q)) return 0;  // 队满，入队失败
           Q.data[Q.rear] = e;
           Q.rear = (Q.rear + 1) % MAXSIZE;  // 队尾指针环形后移
           return 1;
       }

       ```

出队和入队的题目期末考试必考  
- **出队**：队头指针后移（环形），删除元素。  
`c int DeQueue(SqQueue &Q, int &e) { if (QueueEmpty(Q)) return 0; // 队空，出队失败 e = Q.data[Q.front]; Q.front = (Q.front + 1) % MAXSIZE; // 队头指针环形后移 return 1; }`   
- **求队列长度**：  
`c int QueueLength(SqQueue Q) { return (Q.rear - Q.front + MAXSIZE) % MAXSIZE; }`

3. **链式存储结构实现（链队列）**  
    - **定义**：用单链表实现，队头指针指向链表头部（便于删除），队尾指针指向链表尾部（便于插入），通常增加头结点简化操作。

   ```
   // 队列结点结构
   typedef struct QNode {
       int data;
       struct QNode *next;
   } QNode, *QueuePtr;
   // 链队列结构（包含队头、队尾指针）
   typedef struct {
       QueuePtr front;  // 队头指针（指向头结点）
       QueuePtr rear;   // 队尾指针（指向队尾元素）
   } LinkQueue;

   ```

   * **核心操作**
     + **初始化**：创建头结点，队头、队尾指针均指向头结点。

     ```
     void InitQueue(LinkQueue &Q) {
         Q.front = Q.rear = new QNode;
         Q.front->next = NULL;  // 头结点无数据，指针置空
     }

     ```

     + **入队**：新结点插入队尾。

     ```
     int EnQueue(LinkQueue &Q, int e) {
         QNode *p = new QNode;
         p->data = e;
         p->next = NULL;
         Q.rear->next = p;  // 原队尾指向新结点
         Q.rear = p;        // 队尾指针更新为新结点
         return 1;
     }

     ```

     + **出队**：删除队头元素（头结点的下一个结点）。

     ```
     int DeQueue(LinkQueue &Q, int &e) {
         if (Q.front == Q.rear) return 0;  // 队空，出队失败
         QNode *p = Q.front->next;         // 暂存队头元素结点
         e = p->data;
         Q.front->next = p->next;          // 头结点指向新队头
         // 若删除的是最后一个元素，队尾指针需指向头结点
         if (Q.rear == p) Q.rear = Q.front;
         delete p;
         return 1;
     }

     ```

### 四、字符串

#### 4.1 基本概念

* **字符串（String）**：由零个或多个字符组成的有限序列，记为 `S = "c1c2...cn"`（n≥0，n为串长，n=0时为空串）。
* **关键术语**：
  + **子串**：串中任意连续字符组成的序列（如"abc"的子串有"a"、"ab"等）；
  + **主串**：包含子串的串；
  + **模式匹配**：在主串中查找与模式串（待查找的子串）相等的子串的过程。

#### 4.2 存储结构

1. **顺序存储（定长数组）**：用固定大小的数组存储，串长受数组容量限制，需额外记录串长（或用特殊字符`'\0'`作为结束标志，如C语言字符串）。

   ```
   #define MAXLEN 255
   typedef struct {
       char ch[MAXLEN];  // 存储字符串
       int length;       // 串的实际长度
   } SString;

   ```
2. **堆分配存储（动态数组）**：用动态内存分配的数组存储，串长可动态调整（需手动管理内存）。

   ```
   typedef struct {
       char *ch;   // 指向动态分配的字符数组
       int length; // 串长
   } HString;
   // 初始化示例
   void InitString(HString &S) {
       S.ch = NULL;
       S.length = 0;
   }

   ```
3. **链式存储（串链）**：用单链表存储，每个结点存储1个或多个字符（存储1个字符时存储密度低，通常存储多个字符优化）。

   ```
   #define CHUNKSIZE 80  // 每个结点存储80个字符
   typedef struct Chunk {
       char ch[CHUNKSIZE];
       struct Chunk *next;
   } Chunk;
   typedef struct {
       Chunk *head, *tail;  // 串的头、尾指针
       int length;          // 串长
   } LString;

   ```

#### 4.3 模式匹配算法（必考）

1. **暴力匹配（BF算法）**：

   * **思路**：从主串`S`的第1个字符开始，与模式串`T`的第1个字符比较；若相等则继续比较下一个，若不等则主串回溯到上一轮起始位置+1，模式串回溯到第1个字符，重复直至匹配或主串遍历结束。
   * **时间复杂度**：最坏O(n\*m)（n为主串长，m为模式串长，如`S="aaaaaab"`、`T="aaab"`），平均O(n+m)。
   * **核心代码**：

   ```
   // 返回模式串T在主串S中首次出现的起始位置（从1开始），失败返回0
   int Index_BF(SString S, SString T) {
       int i = 1, j = 1;  // i为主串指针，j为模式串指针
       while (i <= S.length && j <= T.length) {
           if (S.ch[i-1] == T.ch[j-1]) {  // 数组下标从0开始，串位置从1开始
               i++; j++;
           } else {
               i = i - j + 2;  // 主串回溯
               j = 1;          // 模式串回溯
           }
       }
       return j > T.length ? i - T.length : 0;
   }

   ```
2. **KMP算法**：

   * **改进点**：避免主串回溯，仅通过模式串的“部分匹配表（next数组）”确定模式串回溯位置，减少比较次数。
   * **next数组**：对模式串`T`，`next[j]`表示`T[1..j-1]`中“最长相等前后缀”的长度（前缀：从第一个字符开始的子串，后缀：以最后一个字符结束的子串）。
     + **示例**：`T="ababc"`，`next[3]=1`（`T[1..2]="ab"`的最长相等前后缀为"a"，长度1）。
     + **next数组求解代码**：

     ```
     void GetNext(SString T, int next[]) {
         int j = 0, k = -1;
         next[0] = -1;  // 初始化（j从0开始，对应模式串第1个字符）
         while (j < T.length - 1) {
             if (k == -1 || T.ch[j] == T.ch[k]) {
                 j++; k++;
                 next[j] = k;
             } else {
                 k = next[k];  // 模式串回溯
             }
         }
     }

     ```
   * **KMP匹配代码**：

   ```
   int Index_KMP(SString S, SString T, int next[]) {
       int i = 0, j = 0;  // 均从0开始（数组下标）
       while (i < S.length && j < T.length) {
           if (j == -1 || S.ch[i] == T.ch[j]) {
               i++; j++;
           } else {
               j = next[j];  // 仅模式串回溯
           }
       }
       return j >= T.length ? i - T.length + 1 : 0;  // 返回1-based起始位置
   }

   ```

   * **时间复杂度**：O(n+m)（预处理next数组O(m)，匹配O(n)），适用于主串和模式串较长的场景。

### 五、树与二叉树

#### 5.1 树的基本概念

* **定义**：n（n≥0）个结点的有限集合，n=0时为空树；n>0时存在唯一“根结点”，其余结点分为若干个互不相交的子树（每个子树也是树）。
* **关键术语**：
  + **父/子结点**：直接相邻的上层/下层结点；
  + **兄弟结点**：同一父结点的子结点；
  + **深度**：从根到该结点的路径长度（根深度为1）；
  + **高度**：从该结点到最远叶子的路径长度（叶子高度为1，树的高度即根的高度）；
  + **度**：结点的子树个数（树的度即所有结点度的最大值）。

#### 5.2 二叉树的定义与性质

1. **定义**：每个结点最多有2个子树（左子树、右子树），子树有左右顺序之分（如“左子树为空、右子树非空”与“右子树为空、左子树非空”是不同二叉树）。
2. **特殊二叉树**：
   * **满二叉树**：所有叶子在同一层，且非叶子结点均有2个子结点（结点数为`2^h - 1`，h为高度）；
   * **完全二叉树**：按层序编号（从1开始），每个结点的编号与满二叉树中对应位置的编号一致（叶子仅在最后两层，且最后一层叶子靠左）。
3. **重要性质（必考）**：
   * 性质1：第i层最多有`2^(i-1)`个结点（i≥1）；
   * 性质2：高度为h的二叉树最多有`2^h - 1`个结点；
   * 性质3：任意二叉树中，叶子结点数`n0` = 度为2的结点数`n2` + 1（`n0 = n2 + 1`）；
   * 性质4：具有n个结点的完全二叉树，高度`h = ⌊log2n⌋ + 1`（或`h = ⌈log2(n+1)⌉`）；
   * 性质5：完全二叉树中，若结点编号为i（1-based），则：
     + 左孩子编号：`2i`（若`2i ≤ n`）；
     + 右孩子编号：`2i+1`（若`2i+1 ≤ n`）；
     + 父结点编号：`⌊i/2⌋`（若`i > 1`）。

#### 5.3 二叉树的存储结构

1. **顺序存储**：适用于完全二叉树，用数组存储，结点按层序编号存入对应下标（非完全二叉树需用“空标记”填充，浪费空间）。

   ```
   #define MAXSIZE 100
   typedef TElemType SqBiTree[MAXSIZE];  // TElemType为结点数据类型（如int）
   SqBiTree T;  // T[i]存储编号为i的结点（T[0] unused或存树的高度）

   ```
2. **链式存储（二叉链表）**：每个结点包含数据域、左孩子指针、右孩子指针（最常用的存储方式）。

   ```
   typedef struct BiTNode {
       TElemType data;                // 数据域
       struct BiTNode *lchild, *rchild;  // 左、右孩子指针
   } BiTNode, *BiTree;

   ```

#### 5.4 二叉树的遍历（必考）

遍历：按一定顺序访问二叉树中所有结点，且每个结点仅访问一次（核心考点：递归/非递归实现、遍历序列还原二叉树）。

1. **递归遍历（简单，需掌握）**

   * **前序遍历（根→左→右）**：

   ```
   void PreOrderTraverse(BiTree T) {
       if (T) {
           visit(T);                // 访问根结点
           PreOrderTraverse(T->lchild);  // 遍历左子树
           PreOrderTraverse(T->rchild);  // 遍历右子树
       }
   }

   ```

   * **中序遍历（左→根→右）**：

   ```
   void InOrderTraverse(BiTree T) {
       if (T) {
           InOrderTraverse(T->lchild);   // 遍历左子树
           visit(T);                // 访问根结点
           InOrderTraverse(T->rchild);   // 遍历右子树
       }
   }

   ```

   * **后序遍历（左→右→根）**：

   ```
   void PostOrderTraverse(BiTree T) {
       if (T) {
           PostOrderTraverse(T->lchild);  // 遍历左子树
           PostOrderTraverse(T->rchild);  // 遍历右子树
           visit(T);                // 访问根结点
       }
   }

   ```
2. **非递归遍历（重点，用栈实现）**

   * **中序遍历（非递归）**：

   ```
   void InOrderTraverse_NonRec(BiTree T) {
       SqStack S;  // 栈存储二叉树结点指针
       InitStack(S);
       BiTree p = T;
       while (p || !StackEmpty(S)) {
           if (p) {                // 左子树遍历到底
               Push(S, p);
               p = p->lchild;
           } else {                // 左子树遍历完，访问根+遍历右子树
               Pop(S, p);
               visit(p);
               p = p->rchild;
           }
       }
   }

   ```
3. **层序遍历（用队列实现）**：按层访问，先访问第1层（根），再第2层，依次类推。

   ```
   void LevelOrderTraverse(BiTree T) {
       LinkQueue Q;  // 队列存储二叉树结点指针
       InitQueue(Q);
       BiTree p;
       if (T) EnQueue(Q, T);  // 根结点入队
       while (!QueueEmpty(Q)) {
           DeQueue(Q, p);      // 出队队头结点
           visit(p);
           if (p->lchild) EnQueue(Q, p->lchild);  // 左孩子入队
           if (p->rchild) EnQueue(Q, p->rchild);  // 右孩子入队
       }
   }

   ```

#### 5.5 二叉树的构造（根据遍历序列）

* **核心规律**：需至少包含“中序遍历序列”+“前序/后序/层序序列”才能唯一确定二叉树（前序/后序/层序序列确定根位置，中序序列区分左、右子树）。
* **示例**：已知前序`[1,2,4,5,3,6]`、中序`[4,2,5,1,6,3]`，构造步骤：
  1. 前序首元素`1`为根，中序中`1`左侧`[4,2,5]`为左子树，右侧`[6,3]`为右子树；
  2. 左子树前序`[2,4,5]`，首元素`2`为左子树根，中序`[4]`为`2`的左子树，`[5]`为右子树；
  3. 右子树前序`[3,6]`，首元素`3`为右子树根，中序`[6]`为`3`的左子树。

#### 5.6 线索二叉树（选考，理解概念）

* **目的**：利用二叉链表中“空的左/右孩子指针”（n个结点的二叉链表有n+1个空指针），存储遍历序列的前驱/后继信息，便于快速查找前驱/后继，无需遍历。
* **分类**：
  + 前序线索二叉树：空指针存储前序遍历的前驱/后继；
  + 中序线索二叉树：空指针存储中序遍历的前驱/后继（最常用）；
  + 后序线索二叉树：空指针存储后序遍历的前驱/后继。
* **结构**：结点增加“左/右线索标志”（0：指向孩子，1：指向前驱/后继）：

  ```
  typedef struct ThreadNode {
      TElemType data;
      struct ThreadNode *lchild, *rchild;
      int ltag, rtag;  // 0：孩子指针，1：线索指针
  } ThreadNode, *ThreadTree;

  ```

### 六、树与森林

#### 6.1 树的存储结构

1. **双亲表示法**：用数组存储，每个结点记录“数据”和“父结点下标”，便于查找父结点，不便于查找子结点。

   ```
   #define MAX_TREE_SIZE 100
   typedef struct {
       TElemType data;
       int parent;  // 父结点下标（根的parent为-1）
   } PTNode;
   typedef struct {
       PTNode nodes[MAX_TREE_SIZE];
       int r, n;  // 根的下标r，结点总数n
   } PTree;

   ```
2. **孩子表示法**：数组+链表，每个结点用链表存储所有子结点，便于查找子结点。
3. **孩子兄弟表示法**：用二叉链表存储，每个结点包含“数据”、“第一个孩子指针”、“右兄弟指针”（将树转为二叉树，便于用二叉树的算法处理树）。

   ```
   typedef struct CSNode {
       TElemType data;
       struct CSNode *firstchild, *rightsib;  // 第一个孩子、右兄弟
   } CSNode, *CSTree;

   ```

#### 6.2 树、森林与二叉树的转换（必考，核心规律）

1. **树转二叉树**：

   * 步骤1：同一父结点的子结点之间用“右兄弟指针”连接；
   * 步骤2：每个结点仅保留“第一个孩子指针”，删除其他子结点指针；
   * 结果：树的根结点无右子树，左子树为原树的子树，右子树为原树的兄弟结点。
2. **森林转二叉树**：

   * 步骤1：将森林中每棵树转为二叉树；
   * 步骤2：将第二棵树作为第一棵树的右子树，第三棵树作为第二棵树的右子树，依次连接。
3. **二叉树转树/森林**：

   * 若二叉树结点有右子树，则右子树为该结点的“兄弟”，需将所有右子树依次作为“兄弟链”连接；
   * 若根结点有右子树，则为森林（多棵树），否则为单树。

#### 6.3 树与森林的遍历

1. **树的遍历**：

   * **先根遍历**：根→依次先根遍历各子树（与对应二叉树的前序遍历一致）；
   * **后根遍历**：依次后根遍历各子树→根（与对应二叉树的中序遍历一致）；
   * **层序遍历**：按层访问（与二叉树层序遍历一致）。
2. **森林的遍历**：

   * **先序遍历**：依次先根遍历森林中每棵树（与对应二叉树的前序遍历一致）；
   * **中序遍历**：依次后根遍历森林中每棵树（与对应二叉树的中序遍历一致）。

### 七、图

#### 7.1 图的基本概念

* **定义**：由顶点集`V`和边集`E`组成，记为`G=(V,E)`，边是顶点之间的关系（无向边/有向边）。
* **关键术语**：
  + **无向图/有向图**：边无方向/有方向（有向边称为弧，记为`<u,v>`，u为弧尾，v为弧头）；
  + **完全图**：无向完全图有`n(n-1)/2`条边（每对顶点间有边），有向完全图有`n(n-1)`条边（每对顶点间有两条方向相反的弧）；就是乘以二倍的关系，画个图一目了然
  + **度**：无向图中顶点的边数（度=入度+出度），有向图中“入度”是指向该顶点的弧数，“出度”是从该顶点出发的弧数；
  + **连通图/强连通图**：无向图中任意两顶点连通（有路径）为连通图；有向图中任意两顶点双向连通（互有路径）为强连通图；
  + **生成树**：连通图的极小连通子图，包含所有顶点，边数为`n-1`（n为顶点数），删除一条边则不连通，增加一条边则有环。

#### 7.2 图的存储结构（必考）

1. **邻接矩阵（顺序存储）**：用`n×n`矩阵`A`存储，`A[i][j]`表示顶点`i`与`j`的关系（无向图：`A[i][j]=A[j][i]`，1表示有边，0表示无边；有向图：`A[i][j]=1`表示有弧`<i,j>`）。

   ```
   #define MAX_VERTEX_NUM 100
   typedef struct {
       VertexType vexs[MAX_VERTEX_NUM];  // 顶点数组
       int arcs[MAX_VERTEX_NUM][MAX_VERTEX_NUM];  // 邻接矩阵
       int vexnum, arcnum;  // 顶点数、边数
   } MGraph;

   ```

   * **特点**：便于判断两顶点是否有边，便于计算度；但空间复杂度O(n²)，不适用于稀疏图（边数远小于n²）。
2. **邻接表（链式存储）**：数组+链表，每个顶点对应一个链表，链表存储该顶点的所有邻接顶点（无向图每条边存储两次，有向图每条弧存储一次）。

   ```
   // 边结点结构
   typedef struct ArcNode {
       int adjvex;  // 邻接顶点的下标
       struct ArcNode *nextarc;  // 下一个邻接边
       InfoType info;  // 边的附加信息（如权值）
   } ArcNode;
   // 顶点结构
   typedef struct VNode {
       VertexType data;  // 顶点数据
       ArcNode *firstarc;  // 指向第一个邻接边结点
   } VNode, AdjList[MAX_VERTEX_NUM];
   // 邻接表结构
   typedef struct {
       AdjList vertices;
       int vexnum, arcnum;  // 顶点数、边数
   } ALGraph;

   ```

   * **特点**：空间复杂度O(n+e)（e为边数），适用于稀疏图；便于查找邻接顶点，不便于判断两顶点是否有边。

#### 7.3 图的遍历（必考，核心算法）

遍历：从指定顶点出发，按一定顺序访问所有顶点，每个顶点仅访问一次（分为深度优先DFS和广度优先BFS）。

1. **深度优先搜索（DFS，类似树的前序遍历）**

   * **思路**：从起点出发，优先访问“未访问的邻接顶点”，递归深入，无法深入时回溯，直至所有顶点访问完毕。
   * **核心代码（邻接表实现）**：

   ```
   bool visited[MAX_VERTEX_NUM];  // 标记顶点是否已访问

   void DFS(ALGraph G, int v) {
       visit(v);                // 访问顶点v
       visited[v] = true;       // 标记为已访问
       ArcNode *p = G.vertices[v].firstarc;  // 指向v的第一个邻接顶点
       while (p) {
           if (!visited[p->adjvex]) {  // 若邻接顶点未访问，递归访问
               DFS(G, p->adjvex);
           }
           p = p->nextarc;  // 访问下一个邻接顶点
       }
   }

   // 遍历整个图（处理非连通图）
   void DFSTraverse(ALGraph G) {
       memset(visited, false, sizeof(visited));  // 初始化访问标记
       for (int v = 0; v < G.vexnum; v++) {
           if (!visited[v]) {  // 未访问的顶点作为起点
               DFS(G, v);
           }
       }
   }

   ```
2. **广度优先搜索（BFS，类似树的层序遍历）**

   * **思路**：从起点出发，先访问起点的所有邻接顶点（第一层），再依次访问每个邻接顶点的邻接顶点（第二层），直至所有顶点访问完毕（用队列实现“层”的管理）。
   * **核心代码（邻接表实现）**：

   ```
   void BFSTraverse(ALGraph G) {
       memset(visited, false, sizeof(visited));
       LinkQueue Q;
       InitQueue(Q);
       for (int v = 0; v < G.vexnum; v++) {
           if (!visited[v]) {
               visit(v);
               visited[v] = true;
               EnQueue(Q, v);  // 起点入队
               while (!QueueEmpty(Q)) {
                   DeQueue(Q, v);  // 出队顶点u
                   ArcNode *p = G.vertices[v].firstarc;
                   while (p) {  // 访问u的所有邻接顶点
                       if (!visited[p->adjvex]) {
                           visit(p->adjvex);
                           visited[p->adjvex] = true;
                           EnQueue(Q, p->adjvex);  // 邻接顶点入队
                       }
                       p = p->nextarc;
                   }
               }
           }
       }
   }

   ```

#### 7.4 最小生成树（MST，必考，针对无向连通图）

* **定义**：在所有生成树中，边的权值之和最小的生成树（适用于“线路规划”等场景，如铺设光缆使所有城市连通且成本最低）。
* **两种核心算法**：

1. **Prim算法**：

   * **思路**：从任意顶点开始，每次选择“与当前生成树连通的、权值最小的边”，将对应的顶点加入生成树，直至所有顶点加入。
   * **时间复杂度**：O(n²)（邻接矩阵），适用于稠密图（边多）。
2. **Kruskal算法**：

   * **思路**：将所有边按权值从小到大排序，依次选择“不构成环的边”加入生成树，直至选择n-1条边（需用“并查集”判断是否构成环）。
   * **时间复杂度**：O(e log e)（排序边的时间），适用于稀疏图（边少）。

#### 7.5 最短路径（必考，针对有向图/无向图）

* **定义**：从起点到终点的路径中，边的权值之和最小的路径（适用于“导航路线规划”等场景）。
* **两种核心算法**：

1. **Dijkstra算法**：

   * **适用场景**：边权值非负的图，求“单源最短路径”（从一个起点到所有其他顶点的最短路径）。
   * **思路**：维护两个集合（已确定最短路径的顶点集S，未确定的顶点集U），每次从U中选择“到起点距离最小的顶点”加入S，并用该顶点更新U中顶点的距离，直至S包含所有顶点。
   * **时间复杂度**：O(n²)（邻接矩阵）。
2. **Floyd算法**：

   * **适用场景**：任意边权值（可负，无负环）的图，求“多源最短路径”（所有顶点对之间的最短路径）。
   * **思路**：动态规划思想，用`dist[i][j]`表示i到j的最短距离，通过中间顶点k更新`dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])`，遍历所有中间顶点k。
   * **时间复杂度**：O(n³)，适用于顶点数较少的图。

#### 7.6 拓扑排序（必考，针对有向无环图DAG）

* **定义**：对DAG的顶点进行排序，使得对于任意有向边`<u,v>`，u在排序序列中位于v之前（适用于“任务调度”，如课程先修关系）。
* **实现步骤（基于入度）**：
  1. 计算所有顶点的入度，将入度为0的顶点入队；
  2. 出队顶点u，加入拓扑序列，删除u的所有出边（即减小对应邻接顶点的入度）；
  3. 若邻接顶点入度变为0，入队；
  4. 重复步骤2-3，直至队空。若拓扑序列长度等于顶点数，则成功；否则图有环，无法拓扑排序。
* **核心代码（邻接表实现）**：

```
int TopologicalSort(ALGraph G) {
    int inDegree[MAX_VERTEX_NUM] = {0};
    int topo[MAX_VERTEX_NUM], count = 0;  // 拓扑序列，序列长度
    LinkQueue Q;
    InitQueue(Q);

    // 1. 计算所有顶点的入度
    for (int v = 0; v < G.vexnum; v++) {
        ArcNode *p = G.vertices[v].firstarc;
        while (p) {
            inDegree[p->adjvex]++;
            p = p->nextarc;
        }
    }

    // 2. 入度为0的顶点入队
    for (int v = 0; v < G.vexnum; v++) {
        if (inDegree[v] == 0) {
            EnQueue(Q, v);
        }
    }

    // 3. 拓扑排序
    while (!QueueEmpty(Q)) {
        DeQueue(Q, v);
        topo[count++] = v;  // 加入拓扑序列
        ArcNode *p = G.vertices[v].firstarc;
        while (p) {  // 删除v的出边，更新邻接顶点入度
            if (--inDegree[p->adjvex] == 0) {
                EnQueue(Q, p->adjvex);
            }
            p = p->nextarc;
        }
    }

    if (count != G.vexnum) {
        printf("图有环，无法拓扑排序\n");
        return 0;
    } else {
        // 输出拓扑序列
        for (int i = 0; i < count; i++) {
            printf("%d ", topo[i]);
        }
        return 1;
    }
}

```

### 八、查找

#### 8.1 基本概念

* **查找**：在数据集合中寻找满足条件的元素（通常按“关键字”查找，关键字是唯一标识元素的数据项）。
* **查找成功/失败**：找到满足条件的元素为成功，否则为失败。
* **平均查找长度（ASL）**：所有查找情况下的查找次数的平均值，衡量查找算法效率的核心指标：
  + 查找成功ASL：`ASL_success = Σ(pi * ci)`（pi为第i个元素的查找概率，ci为查找第i个元素的比较次数）；
  + 查找失败ASL：`ASL_failure = Σ(qj * cj)`（qj为第j个失败区间的查找概率，cj为该区间的比较次数）。

#### 8.2 线性查找（顺序查找）

* **思路**：从数据集合的一端开始，依次与关键字比较，直至找到或遍历结束。
* **适用场景**：顺序存储/链式存储的线性表（无序）。
* **核心代码（顺序表）**：

```
int SequentialSearch(SSTable ST, KeyType key) {
    ST.elem[0].key = key;  // 哨兵，避免越界判断（从后往前找）
    int i = ST.length;
    while (ST.elem[i].key != key) {
        i--;
    }
    return i;  // 找到返回下标，失败返回0（哨兵位置）
}

```

* **ASL**：
  + 查找成功：`(n+1)/2`（概率均等）；
  + 查找失败：`n+1`；
* **时间复杂度**：O(n)。

#### 8.3 折半查找（二分查找）

* **适用场景**：**有序的顺序表**（必须顺序存储，便于随机存取；必须有序，否则无法折半）。
* **思路**：每次将查找区间缩小一半（取中间元素与关键字比较，若相等则成功；若关键字小则在左半区间查找；若大则在右半区间查找）。
* **核心代码**：

```
int BinarySearch(SSTable ST, KeyType key) {
    int low = 1, high = ST.length;  // 1-based下标
    while (low <= high) {
        int mid = (low + high) / 2;  // 中间位置
        if (ST.elem[mid].key == key) {
            return mid;  // 查找成功，返回下标
        } else if (ST.elem[mid].key > key) {
            high = mid - 1;  // 关键字在左半区间
        } else {
            low = mid + 1;   // 关键字在右半区间
        }
    }
    return 0;  // 查找失败
}

```

* **ASL**：`log2(n+1) - 1`（n为元素个数，查找成功，概率均等），如n=10时ASL≈3.1。
* **时间复杂度**：O(log n)。
* **查找判定树**：折半查找的过程可表示为二叉树（中间元素为根，左半区间为左子树，右半区间为右子树），查找次数等于对应结点的深度。

#### 8.4 分块查找（索引顺序查找）

* **思路**：将有序表分为若干“块”（块内无序，块间有序），建立“索引表”（每个块的最大关键字+块的起始/结束下标），查找时先查索引表（折半/线性）确定块，再在块内线性查找。
* **ASL**：`ASL_index + ASL_block`（索引表ASL + 块内ASL），如n=100，分10块，每块10个元素，ASL≈(10+1)/2 + (10+1)/2=11。
* **时间复杂度**：O(√n)（介于线性查找和折半查找之间）。

#### 8.5 树表查找（二叉排序树、平衡二叉树）

1. **二叉排序树（BST，二叉查找树）**

   * **定义**：左子树所有结点关键字 < 根结点关键字；右子树所有结点关键字 > 根结点关键字；左、右子树也为二叉排序树。
   * **核心操作**
     + **插入**：若树空，新结点为根；否则与根比较，小则插入左子树，大则插入右子树（无相等结点，避免重复）。

     ```
     int InsertBST(BiTree &T, KeyType key) {
         if (!T) {  // 树空，插入新结点
             T = new BiTNode;
             T->key = key;
             T->lchild = T->rchild = NULL;
             return 1;
         } else if (key == T->key) {
             return 0;  // 关键字存在，插入失败
         } else if (key < T->key) {
             return InsertBST(T->lchild, key);  // 插入左子树
         } else {
             return InsertBST(T->rchild, key);  // 插入右子树
         }
     }

     ```

     + **查找**：与插入逻辑一致，比较关键字，小则查左子树，大则查右子树，相等则成功。
     + **删除**：需保证删除后仍为二叉排序树，分三种情况：
       1. 叶子结点：直接删除；
       2. 结点仅有左/右子树：用左/右子树替代该结点；
       3. 结点有左、右子树：用“左子树最大结点”或“右子树最小结点”替代该结点，再删除替代结点。
   * **ASL**：取决于树的高度，理想情况（平衡树）O(log n)，最坏情况（单支树）O(n)（如插入有序序列）。
2. **平衡二叉树（AVL树）**

   * **定义**：二叉排序树中，每个结点的“左、右子树高度差（平衡因子BF）”的绝对值≤1（BF=左子树高度-右子树高度，取值-1、0、1）。
   * **目的**：避免二叉排序树退化为单支树，保证查找效率稳定在O(log n)。
   * **失衡与调整**：插入/删除结点后，若平衡因子绝对值>1，需通过“旋转”调整：
     + **LL型**：根的左子树的左子树过高，右旋根结点；
     + **RR型**：根的右子树的右子树过高，左旋根结点；
     + **LR型**：根的左子树的右子树过高，先左旋左子树，再右旋根结点；
     + **RL型**：根的右子树的左子树过高，先右旋右子树，再左旋根结点。

#### 8.6 哈希查找（散列查找）

* **核心思想**：通过“哈希函数”将关键字直接映射到存储地址，实现“快速查找”（理想情况下查找时间O(1)）。
* **关键概念**：
  + **哈希函数（散列函数）**：`addr = H(key)`，将关键字key映射到地址addr；
  + **哈希冲突**：不同关键字映射到同一地址（无法完全避免，需处理）；
  + **哈希表**：存储哈希函数映射结果的数组。

1. **常用哈希函数**：

   * 直接定址法：`H(key) = a*key + b`（适用于关键字连续）；
   * 除留余数法：`H(key) = key % p`（p为小于等于哈希表长度的最大质数，最常用）；
   * 数字分析法：取关键字中分布均匀的数字作为地址（适用于关键字固定长度）。
2. **哈希冲突处理方法**：

   * **开放定址法**：冲突时，按一定规则在哈希表中寻找下一个空地址（如线性探测：`H_i = (H(key) + i) % m`，i=0,1,…；二次探测：`H_i = (H(key) ± i²) % m`）；
   * **链地址法**：哈希表每个地址对应一个链表，冲突的关键字存入同一链表（最常用，无聚集现象）；
   * **再哈希法**：冲突时用另一个哈希函数计算地址。
3. **哈希表的查找**：

   * 计算关键字的哈希地址，若该地址元素与关键字相等则成功；
   * 若冲突，按冲突处理方法查找下一个地址，直至找到或确认失败。
4. **ASL**：取决于哈希函数、冲突处理方法、装填因子α（α=元素个数/哈希表长度，α越小，冲突越少，ASL越小，通常α取0.6~0.8）。
5. **时间复杂度**：理想O(1)，最坏O(n)。

### 九、排序

#### 9.1 基本概念

* **排序**：将无序的记录序列按关键字从小到大（升序）或从大到小（降序）排列。
* **稳定排序/不稳定排序**：若两个关键字相等的记录，排序后相对位置不变，则为稳定排序；否则为不稳定排序。
* **时间复杂度**：主要考虑“关键字比较次数”和“记录移动次数”，与问题规模n相关。
* **空间复杂度**：排序过程中所需辅助空间的大小。

#### 9.2 插入排序（稳定，O(n²)）

1. **直接插入排序**

   * **思路**：将待排序记录依次插入到已排序的子序列中，直至所有记录插入完毕（类似整理扑克牌）。
   * **核心代码**：

   ```
   void InsertSort(int arr[], int n) {
       int i, j, temp;
       for (i = 1; i < n; i++) {  // 从第2个元素开始插入（i=0为已排序子序列）
           if (arr[i] < arr[i-1]) {  // 当前元素需插入已排序子序列
               temp = arr[i];        // 暂存当前元素
               // 已排序子序列元素后移，找到插入位置
               for (j = i-1; j >= 0 && arr[j] > temp; j--) {
                   arr[j+1] = arr[j];
               }
               arr[j+1] = temp;  // 插入当前元素
           }
       }
   }

   ```

   * **性能**：
     + 时间复杂度：最好O(n)（已有序），最坏O(n²)（逆序），平均O(n²)；
     + 空间复杂度：O(1)（原地排序）；
     + 稳定性：稳定。
2. **折半插入排序**

   * **改进点**：在已排序子序列中，用折半查找确定插入位置，减少比较次数（移动次数不变）。
   * **时间复杂度**：最好O(n log n)，最坏O(n²)，平均O(n²)；
   * **稳定性**：稳定。

#### 9.3 交换排序（O(n²)）

1. **冒泡排序**

   * **思路**：重复遍历序列，每次比较相邻元素，若逆序则交换，直至无交换（“大元素”像气泡一样浮到序列末尾）。
   * **核心代码（优化版，无交换则提前结束）**：

   ```
   void BubbleSort(int arr[], int n) {
       bool flag;  // 标记本次遍历是否有交换
       for (int i = 0; i < n-1; i++) {  // 最多需要n-1趟
           flag = false;
           for (int j = 0; j < n-1-i; j++) {  // 每趟减少i个已排序元素
               if (arr[j] > arr[j+1]) {
                   // 交换元素
                   int temp = arr[j];
                   arr[j] = arr[j+1];
                   arr[j+1] = temp;
                   flag = true;
               }
           }
           if (!flag) break;  // 无交换，序列已有序，提前结束
       }
   }

   ```

   * **性能**：
     + 时间复杂度：最好O(n)（已有序），最坏O(n²)（逆序），平均O(n²)；
     + 空间复杂度：O(1)；
     + 稳定性：稳定。
2. **快速排序（重点，O(n log n)）**

   * **思路**：分治法，选择一个“基准元素”，将序列分为两部分（左部分≤基准，右部分≥基准），再递归排序两部分（基准元素位置固定）。
   * **核心代码（以第一个元素为基准）**：

   ```
   // 划分函数：返回基准元素的最终位置
   int Partition(int arr[], int low, int high) {
       int pivot = arr[low];  // 基准元素（第一个元素）
       while (low < high) {
           // 从右往左找≤基准的元素
           while (low < high && arr[high] >= pivot) high--;
           arr[low] = arr[high];  // 移到左半区
           // 从左往右找≥基准的元素
           while (low < high && arr[low] <= pivot) low++;
           arr[high] = arr[low];  // 移到右半区
       }
       arr[low] = pivot;  // 基准元素放入最终位置
       return low;        // 返回基准位置
   }

   // 快速排序递归函数
   void QuickSort(int arr[], int low, int high) {
       if (low < high) {
           int pivotpos = Partition(arr, low, high);  // 划分
           QuickSort(arr, low, pivotpos-1);           // 排序左半区
           QuickSort(arr, pivotpos+1, high);          // 排序右半区
       }
   }

   ```

   * **性能**：
     + 时间复杂度：最好O(n log n)（基准选择合理，划分均匀），最坏O(n²)（基准为最值，划分不均，如有序序列），平均O(n log n)；
     + 空间复杂度：O(log n)~O(n)（递归栈空间，平均O(log n)，最坏O(n)）；
     + 稳定性：不稳定（基准元素与相等元素交换时会改变相对位置）。
   * **优化**：基准选择“三数取中”（首、尾、中间元素的中位数），避免最坏情况。

#### 9.4 选择排序（O(n²)）

1. **简单选择排序**

   * **思路**：每次从待排序序列中选择“最小元素”，与待排序序列的第一个元素交换，直至排序完成。
   * **核心代码**：

   ```
   void SelectSort(int arr[], int n) {
       int min_idx;
       for (int i = 0; i < n-1; i++) {  // 待排序序列从i开始
           min_idx = i;  // 初始最小元素下标为i
           // 找待排序序列中的最小元素
           for (int j = i+1; j < n; j++) {
               if (arr[j] < arr[min_idx]) {
                   min_idx = j;
               }
           }
           // 交换最小元素与待排序序列第一个元素
           int temp = arr[i];
           arr[i] = arr[min_idx];
           arr[min_idx] = temp;
       }
   }

   ```

   * **性能**：
     + 时间复杂度：O(n²)（无论有序/逆序，比较次数固定为n(n-1)/2）；
     + 空间复杂度：O(1)；
     + 稳定性：不稳定（如序列`[2,2,1]`，第一次选择1与第一个2交换，两个2的相对位置改变）。
2. **堆排序（重点，O(n log n)）**

   * **堆的定义**：完全二叉树，分为大根堆（每个父结点≥子结点）和小根堆（每个父结点≤子结点）。
   * **思路**：
     1. 构建大根堆（将无序序列转为大根堆，根为最大值）；
     2. 交换根（最大值）与堆的最后一个元素，将最大值放入最终位置；
     3. 调整剩余元素为大根堆，重复步骤2，直至排序完成。
   * **核心代码**：

   ```
   // 堆调整：将以i为根的子树调整为大根堆（n为堆的大小）
   void HeapAdjust(int arr[], int i, int n) {
       int temp = arr[i];  // 暂存根元素
       // j为i的左孩子，沿较大孩子向下调整
       for (int j = 2*i + 1; j < n; j = 2*j + 1) {
           if (j+1 < n && arr[j] < arr[j+1]) {  // 找左、右孩子中的较大者
               j++;
           }
           if (temp >= arr[j]) break;  // 根元素≥较大孩子，调整结束
           arr[i] = arr[j];  // 较大孩子上移
           i = j;            // i指向当前孩子位置
       }
       arr[i] = temp;  // 根元素放入最终位置
   }

   // 堆排序
   void HeapSort(int arr[], int n) {
       // 1. 构建大根堆（从最后一个非叶子结点开始调整）
       for (int i = n/2 - 1; i >= 0; i--) {
           HeapAdjust(arr, i, n);
       }
       // 2. 交换根与最后一个元素，调整堆
       for (int i = n-1; i > 0; i--) {
           int temp = arr[0];
           arr[0] = arr[i];
           arr[i] = temp;  // 最大值放入最终位置
           HeapAdjust(arr, 0, i);  // 调整剩余i个元素为大根堆
       }
   }

   ```

   * **性能**：
     + 时间复杂度：O(n log n)（构建堆O(n)，调整堆O(n log n)）；
     + 空间复杂度：O(1)（原地排序）；
     + 稳定性：不稳定。

#### 9.5 归并排序（稳定，O(n log n)）

* **思路**：分治法，将序列分为两个子序列，递归排序子序列，再将两个有序子序列“合并”为一个有序序列（核心是“合并”操作）。
* **核心代码（二路归并排序）**：

```
// 合并两个有序子序列：arr[low..mid]和arr[mid+1..high]
void Merge(int arr[], int low, int mid, int high) {
    int i = low, j = mid+1, k = 0;
    int *temp = new int[high - low + 1];  // 辅助数组存储合并结果
    // 合并两个子序列
    while (i <= mid && j <= high) {
        if (arr[i] <= arr[j]) {
            temp[k++] = arr[i++];
        } else {
            temp[k++] = arr[j++];
        }
    }
    // 复制剩余元素
    while (i <= mid) temp[k++] = arr[i++];
    while (j <= high) temp[k++] = arr[j++];
    // 辅助数组元素复制回原数组
    for (k = 0, i = low; i <= high; k++, i++) {
        arr[i] = temp[k];
    }
    delete[] temp;
}

// 归并排序递归函数
void MergeSort(int arr[], int low, int high) {
    if (low < high) {
        int mid = (low + high) / 2;  // 划分中点
        MergeSort(arr, low, mid);    // 排序左子序列
        MergeSort(arr, mid+1, high); // 排序右子序列
        Merge(arr, low, mid, high);  // 合并两个有序子序列
    }
}

```

* **性能**：
  + 时间复杂度：O(n log n)（分治+合并，无论有序/逆序）；
  + 空间复杂度：O(n)（辅助数组，非原地排序）；
  + 稳定性：稳定（合并时相等元素按左子序列优先，不改变相对位置）。

### 十、常考题型与解题技巧

#### 10.1 选择题/填空题高频考点

* **绪论**：时间/空间复杂度计算（重点区分O(1)、O(log n)、O(n)、O(n log n)、O(n²)）、逻辑结构与存储结构的对应关系。
* **线性表**：顺序表与链表的插入/删除时间复杂度对比、链表的遍历（尤其是双向链表、循环链表）。
* **栈与队列**：栈的后进先出特性（括号匹配、表达式求值）、循环队列的判满/判空条件、队列在BFS中的应用。
* **树与二叉树**：二叉树的性质（尤其是n0 = n2 + 1、完全二叉树的高度计算）、遍历序列还原二叉树、线索二叉树的前驱/后继查找。
* **图**：邻接矩阵与邻接表的存储差异、DFS/BFS的遍历序列、最小生成树（Prim/Kruskal）的边选择、最短路径（Dijkstra/Floyd）的距离更新、拓扑排序的环判断。
* **查找**：折半查找的适用条件与ASL计算、二叉排序树的插入/删除结果、平衡二叉树的旋转类型、哈希冲突的处理方法。
* **排序**：各排序算法的时间复杂度、稳定性对比、堆排序的堆调整过程、快速排序的划分结果。

#### 10.2 编程题高频考点

* **线性表**：单链表的逆序（头插法）、链表的合并（如两个有序链表合并为一个有序链表）、顺序表的查找（如找最大/最小元素）。
* **栈与队列**：用栈实现队列、用队列实现栈、括号匹配算法、中缀表达式转后缀表达式并求值。
* **树与二叉树**：二叉树的递归/非递归遍历（尤其是中序非递归）、二叉树的高度计算、二叉排序树的构建与查找。
* **图**：邻接表的构建、DFS/BFS的实现（需输出遍历序列）、拓扑排序的实现（判断图是否有环）。
* **查找**：折半查找的实现、哈希表的构建与查找（链地址法处理冲突）。
* **排序**：快速排序、堆排序、归并排序的实现（需能独立写出核心代码）、简单排序算法的优化（如冒泡排序的提前结束）。

#### 10.3 解题技巧

1. **复杂度分析**：关注“基本操作的执行次数”，忽略常数项和低阶项，如嵌套循环的内层循环次数与外层变量相关时，需用求和公式计算（如`for(i=1;i<=n;i++) for(j=1;j<=i;j++)`的基本操作次数为n(n+1)/2，复杂度O(n²)）。
2. **二叉树遍历序列还原**：牢记“中序+前序/后序”的还原逻辑，先找根，再分左右子树，递归还原。
3. **图的算法实现**：邻接表是编程题的首选存储方式（空间效率高），DFS用递归或栈，BFS用队列，需注意“访问标记”的初始化与更新。
4. **排序算法选择**：根据题目要求（如稳定性、时间/空间限制）选择算法，如“大规模、不稳定”选快速排序，“大规模、稳定”选归并排序，“空间受限”选堆排序。
