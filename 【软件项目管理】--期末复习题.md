**项目**：为了特定的目标，利用有限资源，在一定的时间内，为了开发产品或服务而开展的一系列相关工作叫做项目。

**(临发不整一目多) 临时性/发起人/不确定性/整体性/一次性/目的性/多变性**

目的性：具有范围、时间、成本、质量等方面的独特目标  
 一次性：不存在两个完全相同的项目  
 整体性：不是一项项孤立活动的堆积  
 临时性：有规定的时间段  
 不确定性：目标的复杂性、实施过程的不可完全预见性  
 多变性：资源需求动态、多变、不确定  
 发起人：主要发起人提供资金

#### 12) 如何理解“系统方法”？ISO9000标准的八项原则之一“管理的系统方法”的含义是什么？

![在这里插入图片描述](https://i-blog.csdnimg.cn/blog_migrate/1c388d6d78c1709487b215092b6c0b96.png#pic_center)

![在这里插入图片描述](https://i-blog.csdnimg.cn/blog_migrate/6dfd413dde687d2b6c40e77e7862f039.png#pic_center)

系统方法是指通过对复杂事物进行整体、系统性分析和控制，达到协调各个部分，提高整个系统的效率、效益和质量的一种方法。它是一种综合性的管理方法，通过建立和维护一个系统，以达到更好地实现组织目标的目的。

在ISO 9000标准的八项原则中，“管理的系统方法”是指：管理应该以系统方法为基础进行，通过系统地管理过程，识别、理解和管理相互依赖的过程，以及相互影响的因素，确保组织的效率和效益。这意味着组织应该通过对整个过程的了解和掌握，采取系统化的方法进行管理，以实现组织目标和使其持续改进。

“管理的系统方法”还包括以下几个方面：

1. 基于事实的决策：管理决策应该基于可靠的数据和信息。
2. 以过程为基础的管理：组织应该将其所有活动视为过程，并应用过程管理方法进行控制和改进。
3. 不断改进：组织应该不断地寻求改进，并持续优化其过程和系统。
4. 关注利益相关方：组织应该考虑和满足其所有利益相关方的需求和期望。

通过应用这些原则，组织可以建立一个高效、稳定和可持续的质量管理体系，以确保其持续发展和不断改进。

### 14. 过程管理与改进

#### 44) 什么是CMMI？

Capability Maturity Model Integration（能力成熟度模型集成）

#### 45) CMMI模型的有哪三种用途？

CMMI（Capability Maturity Model Integration）模型是一种用于软件开发、服务提供和过程改进的成熟度模型，具有以下三种用途：

1. 评估：CMMI模型可用于评估组织的软件开发和服务提供能力。通过评估，可以确定组织在软件开发和服务提供方面的能力水平，找到组织的弱点和瓶颈，并提供改进计划。
2. 指导：CMMI模型提供了一套标准的过程指南和最佳实践，可用于指导组织开发和实施可重复、可预测和可控的过程。这些指南和最佳实践包括组织、项目和过程三个层次，以确保组织的软件开发和服务提供过程可管理、可控和可改进。
3. 改进：CMMI模型提供了一种结构化的方法来改进组织的软件开发和服务提供过程。通过分析组织的现有过程和实践，找到其优点和不足之处，并提供改进计划和行动方案，以提高组织的过程成熟度和能力水平。

总的来说，CMMI模型旨在帮助组织开发和实施可靠、高效和高质量的软件和服务，提高组织的业务绩效和客户满意度。[1](#fn1)

#### 46) CMMI模型的阶段表示法分哪五级？各级的主要特征是什么？

![在这里插入图片描述](https://i-blog.csdnimg.cn/blog_migrate/5ec045b771e73e3f54708203255a463b.png)

CMMI模型的阶段表示法分为五个级别，每个级别都描述了一个组织在过程改进方面的成熟度水平。以下是各级的主要特征：

1. 初始级（Level 1 - Initial）：在这个级别上，组织的过程通常是不可预测的、不稳定的、不可控的，缺乏标准化的过程和一致性的方法，因此产品的质量通常是不稳定的。组织的主要目标是建立一个基本的过程框架来建立一些基本的管理控制。改进重点是确立一个基本的管理结构和过程，以便使得过程可控和可管理。
2. 管理级（Level 2 - Managed）：在这个级别上，组织开始建立一些标准化的过程和程序，并确保这些过程能够在项目中得到应用。组织开始监控这些过程的执行情况，以确保它们是可重复的、可预测的和可控的。组织的主要目标是确保过程的稳定性和一致性。改进重点是建立一个可重复的过程，以确保过程稳定、可控和可管理。
3. 定义级（Level 3 - Defined）：在这个级别上，组织开始建立一个标准化的过程框架，并通过培训和沟通来确保所有的人都理解和遵守这些过程。组织开始优化这些过程，并在组织内部共享最佳实践。组织的主要目标是建立一个优化的、可重复的、可预测的和可控的过程。改进重点是建立一个可管理和可优化的过程框架，以提高过程效率和效果。
4. 量化管理级（Level 4 - Quantitatively Managed）：在这个级别上，组织开始量化和分析过程的执行情况，并利用这些数据来进行过程改进和决策。组织开始建立一些度量和指标，并将其应用于过程的改进和控制。组织的主要目标是建立一个度量和分析的过程，以改进过程的效率和效果。改进重点是建立一个可度量、可预测、可优化的过程框架，以提高过程的效率和效果。
5. 优化级（Level 5 - Optimizing）：在这个级别上，组织开始持续地优化过程，并不断地改进过程的效率和效果。组织开始寻找新的技术和方法，并将其应用于过程改进和创新。

每个级别的主要特征如下：

* 初始级：不可预测，反应性方法，无稳定的过程。
* 管理级：实现了重复性的过程，使用基本的项目管理原则来管理项目。
* 定义级：建立了稳定的过程，并对这些过程进行了定义和文档化。
* 量化管理级：通过量化的方法来管理和优化过程，并实时监控过程的性能。
* 优化级：不断寻求改进，采用先进的技术和方法，以优化过程和系统。

### 15. 零缺陷思想

#### 47) 零缺陷管理是什么意思？

零缺陷管理是一种质量管理方法，旨在确保软件产品在交付给客户之前没有任何缺陷。它要求在整个软件开发生命周期中实施一系列质量控制措施，包括预防性控制、检测性控制和纠正性控制，以确保软件产品的质量和稳定性。

在零缺陷管理中，预防性控制是重点，旨在识别和消除导致软件缺陷的潜在原因。它可以通过使用质量工具和技术来实现，如代码评审、[单元测试](https://edu.csdn.net/cloud/sd_summit?utm_source=glcblog&spm=1001.2101.3001.7020)、集成测试、代码分析、重构等。

检测性控制涉及对软件进行各种测试，以确保其符合规格说明书的要求，并且不会导致任何缺陷。测试可以包括单元测试、[集成测试](https://edu.csdn.net/cloud/sd_summit?utm_source=glcblog&spm=1001.2101.3001.7020)、验收测试、性能测试等。

纠正性控制旨在及时发现和修复已经出现的缺陷，以避免其进一步扩大和影响软件产品的质量。纠正性控制可以包括缺陷跟踪、问题管理、[版本控制](https://edu.csdn.net/cloud/sd_summit?utm_source=glcblog&spm=1001.2101.3001.7020)等。

通过实施零缺陷管理方法，软件开发团队可以提高产品质量和稳定性，减少缺陷率，提高用户满意度，并减少软件开发和维护成本。

#### 48) 克劳士比的“质量是免费的”是什么意思？过去认为质量之所以不能免费的原因是什么？

克劳士比的“质量是免费的”指的是在生产过程中，如果正确地执行了所有的过程步骤，产品将不会存在任何质量问题，从而减少了缺陷修复和再制造等成本，实现了零缺陷生产。因此，克劳士比认为，如果采用了正确的工程和管理方法，保证了质量，那么这种质量是没有成本的。

过去认为质量不能免费的原因是因为传统的质量管理方法通常是基于缺陷检测和修复的。这种方法的缺点在于，它仅仅是对缺陷进行修复，而没有针对缺陷产生的根本原因进行分析和改进。这种方法需要花费大量的成本和时间来处理缺陷，并且缺陷修复往往需要在产品交付后进行，这对企业和客户都会带来很大的损失。

然而，现代质量管理方法强调预防性控制，即在生产过程中尽可能地避免缺陷的产生。这种方法可以在生产过程中使用各种工具和技术来提高产品的质量，从而减少缺陷修复和再制造等成本。这种方法强调质量管理是整个生产过程中的一个关键环节，而不是仅仅是一个后续的环节，它强调了预防性控制的重要性，使得质量成为可以免费的。

#### 49) 试述零缺陷管理思想体系的一个中心、两个基本点、三个需要的基本含义。

* 一个中心：零缺陷管理，要求第一次就把事情做正确。
  + 不让缺陷发生生或流至下道工序或其他岗位，那么工作中就可以减少很多处理缺陷和失误造成的成本，工作质量和工作效率都可以大幅度提高，经济效益也会显著增长。
* 两个基本点：有用的和可信赖的。
  + 有用的是一种结果导向的思维，我们做任何事情首先想到它有用，必须站在客户的角度来审视最终的结果是否有用。
  + 但是，如果做的每件事情都有用，也未必可靠，因此，零缺陷管理既追求有用，又追求可靠。
* 三个需要：客户的需要、员工的需要和供应商的需要。任何一个组织首先要承担的是客户。没有客户，组织就没有存在的意义。这三个需要形成一个价值链。因此，必须统一看待客户、员工和供应商的需要。
