## 考研数据结构复习笔记：线性表核心知识点与解题思路

今天终于啃完了数据结构中线性表的部分，作为考研复习的重要模块，这部分内容看似基础却暗藏玄机。结合教材和真题复盘，整理了一份知识点总结，既是对自己学习的梳理，也希望能给同样备考的同学一些参考。

### 一、线性表的核心定义与逻辑特征

线性表是具有相同数据类型的 n 个数据元素的有限序列，其逻辑结构的核心特征可概括为 “**一对一**”—— 除第一个元素外，每个元素有且仅有一个直接前驱；除最后一个元素外，每个元素有且仅有一个直接后继。

这种逻辑结构决定了它的两大物理实现方式：**顺序存储**（顺序表）和**链式存储**（链表）。在考研中，这两种实现方式的对比分析是高频考点，需要从存储密度、插入删除效率、内存分配方式等维度重点区分。

### 二、顺序表：数组实现的底层逻辑

顺序表通过一组地址连续的存储单元实现，本质上是对数组的封装。其存储结构可表示为：

```
\#define MAXSIZE 100

typedef struct{

&#x20;   ElemType data\[MAXSIZE];

&#x20;   int length;

}SqList;

```

**核心操作与时间复杂度**：

* 随机访问：凭借下标可直接定位元素，时间复杂度为 O (1)，这是顺序表的显著优势。
* 插入操作：需移动插入位置后的所有元素，平均时间复杂度为 O (n)，若插入在表尾则为 O (1)。
* 删除操作：与插入类似，需移动被删元素后的元素，平均时间复杂度为 O (n)。

**考研常见考点**：

* 顺序表的扩容问题，需注意扩容后原数据的复制过程及时间成本。
* 基于顺序表的算法设计，如合并两个有序表、查找主元素等，需熟练掌握双指针技巧。

### 三、链表：指针串联的动态结构

链表通过指针将分散的节点串联，无需连续内存空间，常见类型包括单链表、双链表和循环链表。单链表的节点结构定义为：

```
typedef struct LNode{

&#x20;   ElemType data;

&#x20;   struct LNode \*next;

}LNode,\*LinkList;

```

**核心操作与时间复杂度**：

* 访问元素：需从表头开始遍历，时间复杂度为 O (n)，无法像顺序表那样随机访问。
* 插入 / 删除：只需修改指针指向，时间复杂度为 O (1)，但前提是已找到操作位置（寻找位置的时间为 O (n)）。

**不同链表的对比**：

* 单链表：仅有后继指针，逆向操作需从头遍历，适用于单向访问场景。
* 双链表：同时拥有前驱和后继指针，支持 O (1) 时间的逆向访问，常用于需要频繁双向遍历的场景。
* 循环链表：表尾指针指向表头，可简化首尾元素的操作，如解决约瑟夫环问题。

**考研常见考点**：

* 链表的创建（头插法与尾插法）及区别，头插法可实现数据逆序存储。
* 链表的合并、拆分、环检测（快慢指针法）等算法设计，需画图辅助理解指针变化。

### 四、顺序表与链表的对比及应用场景

| 特性 | 顺序表 | 链表 |
| --- | --- | --- |
| 存储方式 | 连续内存空间 | 离散内存空间，指针连接 |
| 存储密度 | 高（无指针开销） | 低（需额外存储指针） |
| 插入删除效率 | 平均 O (n) | 平均 O (1)（找到位置后） |
| 随机访问能力 | 支持（O (1)） | 不支持（O (n)） |
| 内存分配方式 | 静态分配或动态扩容 | 动态分配 |

**选择原则**：

* 若需频繁随机访问，优先选择顺序表，如数组下标访问场景。
* 若插入删除操作频繁且位置不固定，优先选择链表，如实现链表式队列、栈等。

### 五、考研复习建议与刷题技巧

1. **画图辅助理解**：尤其是链表操作，每一步指针变化都要在图中清晰标注，避免抽象思维导致的错误。
2. **动手实现代码**：不要仅停留在理论层面，亲手编写顺序表和链表的基本操作代码，熟悉边界条件（如空表、表满、表头表尾操作）。
3. **真题分类训练**：按 “基本概念辨析”“算法设计与实现”“时间空间复杂度分析” 三类题型整理真题，总结解题模板。
4. **对比记忆**：通过表格或思维导图梳理顺序表与链表的异同，加深对两种结构适用场景的理解。
